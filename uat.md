# /uat — Human Acceptance Testing (UI Stories Only)

## Purpose

Present the pre-generated UAT checklist (`epic-N-uat.md`, produced by `/prd` at planning time) for the human to manually verify the look, feel, and interaction of all UI stories in the epic on a real device or browser. The human ticks items off in chat. A go-ahead from the human is the gate for merging the epic.

`/uat` does not regenerate the checklist — it loads and presents `epic-N-uat.md`. If the file is missing, `/uat` regenerates it from all `story-N.md` files in the epic and logs a warning.

---

## When /uat Runs

| Epic Type | After /check | Action |
|-----------|-------------|--------|
| UI epic (contains any UI story) | ✅ CI green | Run `/uat` — human must approve before merge |
| Backend only (no UI files touched in any story) | ✅ CI green | Skip `/uat` — merge directly |

**How Claude determines epic type:** Reads the Story Type field for each story in the epic from `Sprint.md`. If any story is UI type → UAT required for the whole epic. If all stories are Backend type → no UAT.

---

## What Claude Generates

Claude loads `epic-N-uat.md` (pre-generated by `/prd`) and presents it directly. It tells the human exactly what to open, what to do, and what to look for — covering all stories and cross-story flows in the epic.

If `epic-N-uat.md` is missing, Claude regenerates it from all `story-N.md` files in the epic, presents it, and logs: "Warning: epic-N-uat.md was not found — regenerated now. Check /prd ran correctly for this epic."

---

## UAT Checklist Format

The checklist is loaded from `epic-N-uat.md` — pre-generated by `/prd`. Format covers all stories in the epic plus cross-story flows. See `epic-N-uat.md` format in `/prd` Phase 5 Output Files for the full template.

Summary of what the checklist covers:
- Per-story flows (happy path + edge cases) for every UI story in the epic
- Visual checks per story (colours, typography, spacing, interactive states, animations)
- Cross-story flows — journeys that span multiple stories
- Responsive / device check across all epic screens
- Sign-off prompt

---

## Human Sign-off in Chat

The human reviews each item on the checklist manually on the real device or browser, then replies in the chat:

| Response | Meaning | Next Step |
|----------|---------|-----------|
| ✅ APPROVED | All items verified, story looks and feels correct | Claude creates PR summary + merge instruction |
| ❌ CHANGES NEEDED | One or more items failed or look wrong | Claude appends the rejection to `uat-log.md`, then loops back to `/dev` for the fix |

---

## After Approval

1. **Confirm PR exists** for the epic branch. Create one if missing.

2. **Merge to main:**
   ```bash
   gh pr merge [PR-number] --squash --delete-branch
   git checkout main && git pull
   ```

3. **Append a checkpoint to `HANDOVER.md`** — written at merge time:
   ```markdown
   ---

   ## Checkpoint: Epic #N [name] | [YYYY-MM-DD] | MERGED ✅

   Epic complete: [one sentence — what the user can now do]
   Stories merged: #[first]–#[last]

   UAT fixes applied: [list any fixes made between /check and UAT sign-off, or "none"]

   Next: Epic #[N+1] [name] | Ready for: /prd
   ```

4. **Update `Sprint.md`:**
   - Mark epic `✅` in Epic & Story Index — all stories within it
   - Update Active Pointers to point to Epic #N+1

5. **Clear `TODO.md`:**
   ```
   # TODO
   _Ready for Epic #N+1: [name] — run /prd to generate task list._
   ```

6. **Commit and push to main:**
   ```bash
   git add HANDOVER.md Sprint.md TODO.md
   git commit -m "chore(epic-N): UAT approved — mark complete, clear TODO for epic #N+1"
   git push origin main
   ```

7. **If this epic had UAT rejections**, update those entries in `uat-log.md`: fill in Fix Applied and set Status to Resolved.

8. **Tell the user:** "Epic #N complete and merged ✅. Ready to run /prd on Epic #[N+1]?"

---

## After Changes Needed

Do not re-run `/uat` immediately. First, append a rejection entry to `uat-log.md` using the format below. Then go back to `/dev` to fix the specific issues in the affected story. Re-run `/check`. Then re-run `/uat` — it will present the existing `epic-N-uat.md` checklist.

```markdown
# uat-log.md — append one block per rejection, never delete entries

---

## UAT Rejection: Epic #N — [Epic Name] | [YYYY-MM-DD HH:MM]

Attempt: [N]   # increment if same epic is rejected multiple times

Failed items:
- [Story #X: checklist item that failed, copied verbatim from epic-N-uat.md]

Human feedback: [exact words the human used]

Root cause (if known): [e.g. wrong token in story #X, missing state, responsive rule not applied]

Fix applied: [filled in after /uat passes — one line per fix]

Status: Open | Resolved
```
